이번 단원은 문제를 푸는것을 위주로 진행되어서, 필기할것보다는 실습할게 많은 단원이였다. 그래서 평소때보다 조금 더 어려웠고 시간도 오래 걸렸다.


🖋 5. 문제 해결 패턴


🖊 5.1 문제 해결 패턴 소개

이전단원에 이어져 있는 단원이다.

1.코드를 작성하기 전에 어떻게 접근할것인지.
2.일반적인 문제 해결패턴을 찾을지.

에서 두번째 내용을 배운다.
사실 암기할 것도 어느정도 있다.

서투른 해결책 -> 순진한 접근법 -> 올바른 해결책의 순서대로 코딩을 진행해본다.


🖊 5.2 빈도수 세기 패턴

이 패턴은 객체를 사용해서 다양한 값과 빈도를 수집해 문제를 해결한다.
루프를 사용하게 되면 O(N^2)의 시간 복잡도가 나오기 때문에 중첩 루프를 배제할 수 있다면 좋은 결과를 얻을 수 있다.

새로 알게된 매소드
* correctIndex
* splice

루프를 두개 중첩시키는 것 보다, 별개의 루프를 두 개 만드는 것이 속도의 복잡도면에서 더 이득을 볼 수있다.

그리고 for of 를 조금 더 정리해봐야할 것 같다.


🖊 5.3 빈도수 세기 : 애너그램 도전 과제

두개의 문자열을 취하며, 두 문자열이 서로의 아나그램이면 참이 반환되는 과제다.
문제 해결방법은 많지만 위의 목표치에 맞춰서 문제를 풀어본다.


🖊 5.4 애너그램 도전 과제 솔루션

우선 놀랐던 것은 웹 자체에 문제를 풀 수있는 기능이 있어서 좋았는데, 한번한번 답을 확인할때마다 조금 답답한 감이 있어 VSC에 input을 옮겨적어 놓고 풀었다.
혹시나 내 글을 보는 사람이 있다면 '쿼카'라는 익스텐션을 추천한당. 왜냐면 바로바로 로그나 리턴값을 확인할수 있거덩.
결론은 문제를 잘 풀지 못했다.


🖊 5.5 다중 포인터 패턴

인덱스나 위치에 해당하는 포인터나 값을 만든 다음 특정 조건에 따라 중간지점에서부터 시작지점이나 끝 지점이나 양쪽지점을 향해 이동시키는 문제의 형태다.
혹은 배열이나 문자열같은 일종의 선형구조나 이중연결리스트나 단일 연결리스트들을 만드는 것의 문제가 나오기도 한다.
중요한 것은 다수의 포인터 값!


🖊 5.5 다중 포인터 : 고유값을 세는 도전 과제

두개의 포인터를 사용해서 각각의 포인터를 옮기고 값을 원래 있던 곳으로 바꿔주면서 문제를 풀어준다.
사실 기존의 방법대로 문제를 풀때는 배열을 바꾼다는 생각을 못해봤는데, 아무래도 함수를 따로 분리하거나 새로운 복제 어레이를 만들 생각을 안해서 그랬던 것 같다.
조금 사고의 폭을 넓혀야 겠다.


🖊 5.6 고유값 세기 솔루션

같은 수가 몇개나 있는지 찾는 문제형태.
위와 같은 상황에서 값이 다른 경우에만 바꿔줌


🖊 5.7 기준점 간 이동 배열 패턴

슬라이딩 윈도우
배열이나 문자열과 같은 일련의 데이터를 입력하거나 특정 방식으로 연속적인 해당 데이터의 하위 집합을 찾는 형태의 문제.
또는 연속된 숫자의 합이 제일 큰 것을 찾는 문제.


🖊 5.8 분할과 정복 패턴

문서화된 이름이라고도 불리는 패턴
분할정복 알고리즘이라서 기본적인 알고리즘보다 복잡하다.
퀵정렬, 병합정렬, 이진탐색 등 나중에 배울것의 기본이 되는 부분이라 간단하게만 설명을 하고 넘어가셨다.

문제를 푸는 전체적인 컨셉은 전체의 예시를 다루지 않고 부분 부분 나눠서 해당하는 내용만 보는 것이다.

이 부분에서 처음으로 O(logN)이 나온다.
이진법에서는 중간을 선택하고 중간을 선택하고 1/2씩 나누면서 배열을 쪼개면서 줄여서 작은 배열중에서만 검색을 하도록 진행하는 방법을 사용하니,
O(N)이 아닌 O(logN)이 나오는 것 같다.

이 이진에 관한 내용은 뒤에서 더 자세히 다룬다고 한다.
